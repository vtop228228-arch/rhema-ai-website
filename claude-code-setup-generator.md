## Два подхода к созданию продакшн агентов

### Подход A: Anthropic Agent SDK (рекомендуется для сложных агентов)
Используй Claude Agent SDK (@anthropic-ai/claude-agent-sdk) когда агенту нужно:
- Выполнять код, работать с файлами
- Использовать MCP-серверы для интеграций
- Выполнять сложные многошаговые задачи
- Координировать субагентов

Архитектура:
```
Сервер (Next.js API Route / Node.js на VPS)
  └── Claude Agent SDK
        ├── Системный промпт (роль, ограничения)
        ├── Инструменты (MCP, custom tools)
        ├── Контекст пользователя (из БД)
        └── Результат → ответ пользователю
```

Установка: npm install @anthropic-ai/claude-agent-sdk
Аутентификация: ANTHROPIC_API_KEY (НЕ claude.ai логин)

### Подход B: Anthropic Messages API (рекомендуется для простых агентов)
Используй прямой API (@anthropic-ai/sdk) когда агенту нужно:
- Отвечать на вопросы пользователей
- Классифицировать, анализировать, генерировать текст
- Вызывать инструменты (Tool Use)
- Работать в режиме чата

Архитектура:
```
Сервер (API Route / Node.js на VPS)
  └── Anthropic SDK → messages.create()
        ├── system: "Ты — агент поддержки компании X..."
        ├── tools: [{ name: "search_docs", ... }]
        ├── messages: [история чата]
        └── Результат → стрим в UI
```

Установка: npm install @anthropic-ai/sdk

## Принципы работы
- Всегда проверяй существующую схему перед изменениями
- Каждое изменение — через миграцию в supabase/migrations/
- Именование таблиц: snake_case, множественное число (users, tickets, orders)
- Каждая таблица обязательно содержит: id (uuid, default gen_random_uuid()), created_at, updated_at
- FK всегда с ON DELETE CASCADE или SET NULL — обосновывай выбор

## RLS (Row Level Security)
- ВКЛЮЧАЙ RLS для каждой таблицы без исключений
- Для пользовательских данных: auth.uid() = user_id
- Для публичных данных: SELECT для всех, INSERT/UPDATE/DELETE для владельца
- Для админских данных: проверяй роль через user_metadata или отдельную таблицу ролей
- Всегда тестируй политики: пиши SQL-запросы для проверки от имени разных ролей

## Формат миграции
- Имя файла: YYYYMMDDHHMMSS_описание.sql
- Каждая миграция — атомарная операция (одна логическая единица)
- Комментарии в SQL объясняют ПОЧЕМУ, а не ЧТО
- В конце миграции — COMMENT ON TABLE/COLUMN для документации

## Что проверяешь
1. Безопасность:
   - RLS политики покрывают все операции (SELECT, INSERT, UPDATE, DELETE)
   - Нет утечки данных между пользователями

   - Граничные случаи (пустые данные, null, undefined)
   - Конкурентный доступ (два пользователя редактируют одно)
   - Обработка ошибок сети, таймаутов

   - N+1 запросы

## ЧАСТЬ 6: MCP-СЕРВЕРЫ

Сгенерируй команды подключения MCP на основе спецификации.

### Обязательные MCP-серверы (подключай ВСЕГДА):

#### Context7 — актуальная документация библиотек
Context7 MCP подтягивает актуальную, версионно-специфичную документацию и примеры кода прямо из официальных источников — и вставляет их непосредственно в контекст Claude Code. Без него Claude пишет код по устаревшим API из своих тренировочных данных, что приводит к ошибкам.

```bash
# Вариант 1: Удалённый сервер (рекомендуется, не нужен Node.js)
claude mcp add --scope user --transport http context7 https://mcp.context7.com/mcp

# Вариант 2: С API-ключом (для повышенных лимитов — бесплатный ключ на context7.com/dashboard)
claude mcp add --scope user --transport http --header "CONTEXT7_API_KEY: ТВОЙ_КЛЮЧ" context7 https://mcp.context7.com/mcp

# Вариант 3: Локальный через npx (нужен Node.js 18+)
claude mcp add context7 -- npx -y @upstash/context7-mcp@latest
```

Как использовать: добавляй `use context7` в конец промпта, когда работаешь с библиотеками:
```
Создай middleware в Next.js 16 для проверки JWT. use context7
Настрой Supabase Auth с email/password. use context7
Создай форму с react-hook-form и Zod-валидацией. use context7
```

ВАЖНО: В CLAUDE.md добавь правило:
```
## Context7
При работе с любыми внешними библиотеками (Next.js, React, Supabase, Tailwind и т.д.)
ВСЕГДА используй Context7 MCP для получения актуальной документации перед написанием кода.
Добавляй "use context7" к запросам, связанным с API библиотек.
```

   - `opus` — для сложных задач (архитектура, БД, безопасность, платежи)

6. **Максимальная конкретика:** Вместо "используй лучшие практики" пиши конкретные правила. Вместо "правильная архитектура" пиши точную структуру папок.
