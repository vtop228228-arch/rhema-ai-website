# SPEC.md — Rhema AI Website

## 1. Обзор стека

| Слой | Технология | Роль |
|---|---|---|
| Framework | Next.js 15 App Router | Роутинг, SSG, SEO |
| Язык | TypeScript 5 strict | Типобезопасность |
| Стили | Tailwind CSS v4 | Утилитарные классы |
| Шрифты | Bebas Neue + Plus Jakarta Sans | Google Fonts через next/font |
| Деплой | Vercel | Хостинг, CI/CD из GitHub |
| Форма | Нативный fetch → Telegram Bot API | Заявки в Telegram |

**Запрещено:** Prisma, Supabase, базы данных, авторизация, платежи (сайт статичный).

## 2. Структура страниц и User Stories

### Страница 1: `/` (Главная)

**US-1:** Как посетитель, я хочу понять за 5 секунд что делает Rhema AI.
- Критерий: Hero с заголовком "ОТ ДИАГНОСТИКИ ДО МАСШТАБИРОВАНИЯ" и подзаголовком.

**US-2:** Как посетитель, я хочу узнать для кого эти услуги.
- Критерий: Секция "Для кого наши услуги" — 5 пунктов списком с cyan-точками.

**US-3:** Как посетитель, я хочу понять что конкретно делает агентство.
- Критерий: Секция "Мы делаем так" с табами: Диагностика | AI-агенты | Разработка ПО | SaaS-платформы.

**US-4:** Как посетитель, я хочу знать примерные цены.
- Критерий: Таблица цен (Услуга / Срок / Стоимость).

**US-5:** Как посетитель, я хочу перейти к подробной информации.
- Критерий: CTA "Узнать подробнее" ведёт на `/how-we-work`.

---

### Страница 2: `/how-we-work` (Как мы работаем — главная продающая)

**US-6:** Как потенциальный клиент, я хочу понять как именно потеряны мои деньги.
- Критерий: Hero "Ваш бизнес теряет 20-40% прибыли. Но вы не видите где." + гарантия.

**US-7:** Как клиент, я хочу убедиться что это про меня.
- Критерий: Секция "Для кого" — 5 сценариев от первого лица.

**US-8:** Как клиент, я хочу понять почему я сам не вижу проблему.
- Критерий: Секция "Мифы" — 4 карточки Миф → Реальность.

**US-9:** Как клиент, я хочу увидеть конкретные потери в рублях.
- Критерий: Секция "Но у меня всё работает" — 3 сценария с суммами потерь.

**US-10:** Как клиент, я хочу понять процесс работы.
- Критерий: Два шага (Диагностика + Внедрение) с деталями.

**US-11:** Как клиент, я хочу знать что происходит каждый день диагностики.
- Критерий: 4 карточки День 1-4 + блок "После диагностики" с процентами.

**US-12:** Как клиент, я хочу увидеть реальные результаты.
- Критерий: 3 кейса в формате "Думал X → Нашли Y → Сделали Z → Результат".

**US-13:** Как клиент, я хочу выбрать подходящий тариф.
- Критерий: 3 тарифа (Экспресс / Полная / Enterprise).

**US-14:** Как клиент, я хочу получить ответы на вопросы.
- Критерий: FAQ — 10 вопросов и ответов с аккордеоном.

**US-15:** Как клиент, я хочу оставить заявку.
- Критерий: Форма (Имя + Telegram/WhatsApp + Описание + 2 чекбокса согласия) → отправка в Telegram.

## 3. Дизайн-система — «Ethereal Glass»

> Источник истины — `src/app/globals.css`. Подробнее — `DESIGN_REFERENCE.md`.
> OLED-чёрный + оранжевый, мягкое свечение, стекло, скруглённые углы.

### Цвета (CSS переменные + `@theme` токены в globals.css)
```css
--bg:     #050505   /* основной фон — OLED-чёрный */
--bg2:    #080808   /* альтернативный фон секций */
--card:   #0d0d0d   /* база карточек */
--card2:  #110e0b   /* карточка с тёплым оттенком */
--line:   rgba(255,255,255,0.07)   /* бордеры */
--line2:  rgba(255,255,255,0.12)   /* активные бордеры */
--accent: #FF5C1A   /* основной акцент — оранжевый */
--accent2:#FF7A3D   /* оранжевый hover */
--green:  #22C55E   /* позитивные результаты */
--red:    #EF4444   /* потери и убытки */
--ink:    #F5F5F5   /* основной текст */
--sub:    #4a4a4a   /* приглушённый */
--sub2:   #888888   /* второстепенный */
--blue:   var(--accent)   /* legacy-алиас → оранжевый */
```

### Типографика
- **Заголовки:** `Bebas Neue`, letter-spacing 0.03em, line-height 1.04
- **Тело:** **Plus Jakarta Sans** (не Inter), font-size 15px, line-height 1.72
- **Метки:** Plus Jakarta 10px, font-weight 700, uppercase, letter-spacing 0.22em, color --accent

### Кнопки
Три варианта, все со скошенным левым краем (clip-path):
```
btn-blue  → оранжевый градиент-fill (--accent→#e84f10), белый текст, оранжевая тень, hover-подъём
btn-line  → border оранжевый полупрозрачный, color --accent, hover-glow
btn-dim   → fill rgba(255,255,255,0.04), border --line2
clip-path: polygon(14px 0%,100% 0%,100% 100%,0% 100%)
```
Размеры: `btn-sm` / `btn-md` / `btn-lg`.

### Секции
- Padding: `112px 0` (десктоп)
- Max-width контейнера: 1120px, padding: 0 32px
- Чередование фона: `--bg` / `--bg2`
- Глобальная атмосфера: `body::before` радиальный оранжевый glow + `body::after` noise grain

### Карточки (Double-Bezel)
- `card-shell`+`card` (премиум), `card-simple` (hover-glow + подъём), `card-accent` (тёплый, featured)
- Градиентный фон `#131313→#0c0c0c`, border `--line`, border-radius `--r`(14px)/`--r-lg`(20px)
- hover: оранжевый бордер + `--glow-sm` + `translateY(-3px)`, верхний световой блик (`::before`)

## 5. API — Отправка заявки в Telegram

**Endpoint:** `POST /api/contact`

**Request body:**
```typescript
{
  name: string       // обязательно
  contact: string    // Telegram или WhatsApp, обязательно
  business: string   // описание бизнеса, обязательно
  consent: boolean   // согласие на обработку данных, должно быть true
}
```

**Логика:**
1. Zod-валидация входных данных
2. Формирование текста сообщения
3. `fetch` к `https://api.telegram.org/bot{TOKEN}/sendMessage`
4. Ответ: `{ ok: true }` или `{ error: string }`

**ENV переменные:**
```
TELEGRAM_BOT_TOKEN=...
TELEGRAM_CHAT_ID=...   # Telegram ID Владислава
```

## 7. Edge Cases

- Форма: disabled кнопка во время отправки + success state
- Форма: если Telegram API недоступен → показать сообщение "Напишите нам напрямую: @rhema_ai"
- Форма: валидация на клиенте (required fields) + серверная Zod-валидация
- FAQ аккордеон: только один открыт одновременно
- Шрифты: `font-display: swap` через next/font чтобы не блочить рендер
- Изображения: placeholder для wireframe-иллюстрации (заменить потом)
- Mobile: nav скрывается, кнопка "Получить диагностику" остаётся
